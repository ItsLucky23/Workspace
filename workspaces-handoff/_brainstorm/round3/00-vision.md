## Vastgelegde scope + wat deze ronde doet

De grote richting staat. Deze ronde gaat één niveau dieper: **concrete bouw- en UX-keuzes** per tool, in de bouwvolgorde **Interviewer → Designer → Marketing-setup → Document**. De vragen nemen de onderstaande, al-gemaakte keuzes als gegeven aan — wijzig je iets daarvan, zet het in het vrije veld onderaan.

**Vastgelegd (scope-ronde):**

- **Raamwerk:** één gedeeld tool-raamwerk (vrije mappenboom + skills + generatie + artifact-store), per workspace aan/uit. Eén skill-model met een `surface`-type; skills centraal in het bestaande skills-tab, gefilterd.
- **Designer:** opgeslagen design = **preview + code**; raakt de repo nooit direct (alleen artifacts); 1 design per geselecteerde skill; vrije plaatsing; vergelijken op de pagina, definitief kiezen bij het ticket; **volle breedte** (pagina's + componenten + design-systeem).
- **Marketing:** V1 = **alleen setup** (pagina + folders + skill-config, geen generatie); root-actie + folders; context = features + screenshots + branding **+ docs/graph/pipeline-context**; externe API's als workspace-integratie.
- **Document:** in de **eerste lichting**; zelfde model, uploads in de folder; **project- én algemene** documenten; **natuurlijke/professionele kwaliteit — geen AI-detectie-ontwijking**.
- **Interviewer:** eigen tool-pagina; output = vragen + idee-kaarten; "maak ticket"-knop per idee; on-demand. **Wordt als eerste gebouwd.**
- **Koppeling:** mens → ticket, met één **opt-in** uitzondering (een ticket mág de Interviewer vragen, per workspace aan, standaard uit); artifact gevonden via picker + semantisch zoeken; **live verwijzing**; Stage-Agent gebruikt het als referentie.

> Deze vragen gaan over *hoe het werkt en aanvoelt* (UX, gedrag, V1-knip) — nog steeds keuzes die jij maakt, niet de pure code-implementatie.
