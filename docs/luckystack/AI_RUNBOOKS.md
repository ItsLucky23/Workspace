## Add an API route

**Goal:** a new endpoint at `api/{page}/{name}/v{N}`, auth-gated.

1. copy the existing `billing/_api/listInvoices_v1.ts` as your starting point.
2. Create `src/{page}/_api/{name}_v1.ts` exporting `httpMethod`, `rateLimit`, `auth`, `ApiParams`, and `main`.
   - Auth: `export const auth: AuthProps = { login: true, additional: [] };` (or `login: false` for public).
   - In `main`, prefer the injected `functions.tryCatch.tryCatch(...)` over raw try/catch (Rule 21).
3. Regenerate types: `npm run generateArtifacts` (emits the route into `apiTypes.generated.ts`).
4. Scaffold a test: `npm run scaffold:test {page}/{name}/v1`, then fill at least one happy-path case.
5. **Verify:** `npm run lint && npm run build`, then `npm run test`. Call it from the client with
   `apiRequest({ name: '{page}/{name}', version: 'v1', data: {...} })` — typing must resolve with no casts.

Full spec: `docs/ARCHITECTURE_API.md`. Browse existing routes: `docs/AI_PROJECT_INDEX.md`.

## Add a page

**Goal:** a new route `/{page}` rendered by `src/{page}/page.tsx`.

1. copy the existing `admin/page.tsx` as your starting point.
2. Create `src/{page}/page.tsx` with a default export component and `export const template = 'plain' | 'dashboard'`.
3. User-facing text MUST go through `useTranslator` (Rule 13) — no hardcoded strings.
4. Colors/classes only from the `@theme` tokens in `src/index.css` (Rule 14) — never arbitrary hex.
5. Reuse `src/_components/` primitives (Dropdown, MenuHandler, Avatar, …) — check the component table in `CLAUDE.md` before building a new one.
6. **Verify:** `npm run lint && npm run build`; load `/{page}` in the dev server.

Routing rules: `docs/ARCHITECTURE_ROUTING.md`.

## Add a real-time sync event

**Goal:** a room-based realtime event `sync/{page}/{name}/v{N}`.

1. copy the existing `playground/_sync/echo_server_v1.ts` as your starting point.
2. Create `_sync/{name}_server_v1.ts` (runs ONCE on the server for validation).
3. Only add `_client_v1.ts` if per-client logic is needed (filtering, per-client auth, custom `clientOutput`). If it would only return `{ status: 'success' }`, do NOT create it.
4. `_client` files receive `token`, not `user` — call `functions.session.getSession(token)` only if needed.
5. `npm run generateArtifacts`, then scaffold + fill a test.
6. **Verify:** client sends `syncRequest({ name, data, receiver: roomCode })` and receives via `upsertSyncEventCallback`.

Full spec: `docs/ARCHITECTURE_SYNC.md`.

## Add a reusable helper or component

**Goal:** shared logic in `src/_functions/` or a UI primitive in `src/_components/`.

1. FIRST check `docs/AI_CAPABILITIES.md` — the helper/component may already exist (Rule 12). Extend it rather than rolling a parallel one.
2. Helper: copy the existing `_functions/confetti.ts` as your starting point. Component: copy the existing `_components/Avatar.tsx` as your starting point.
3. After adding any new export, run `npm run ai:capabilities` (autonomous) so the snapshot sees it.
4. **Verify:** `npm run lint && npm run build`; the new export shows up in `docs/AI_CAPABILITIES.md`.

## Verify a change before declaring done

**Goal:** the change is actually correct, per CLAUDE.md Rule 1a (transform the task into a verifiable goal).

1. `npm run lint && npm run build` — zero warnings, zero errors (Rule 11).
2. `npm run test` — the auto-sweep + your per-route cases. For a bug fix, a regression test must exist and pass.
3. Server-start is a developer action — ask the user to run `npm run server` + `npm run client` for any browser check.
4. Refresh snapshots in-session after structural changes: `npm run ai:project-index` / `npm run ai:capabilities`.
